# 🚀 生产部署超详细教程

## 📋 前提条件检查

在开始之前，请确认：

1. ✅ 你有 GitHub 账号
2. ✅ 你有邮箱（用于注册服务）
3. ✅ 项目代码已推送到 GitHub 仓库
4. ✅ 本地测试都通过

---

## 第一步：将代码推送到 GitHub

### 1.1 如果代码还没推送到 GitHub

```bash
# 在项目根目录执行

# 查看当前修改
git status

# 添加所有修改的文件
git add .

# 提交修改（写一个清晰的提交信息）
git commit -m "Fix CORS configuration and optimize imports"

# 如果还没创建 GitHub 仓库，先在 GitHub 网站创建一个新仓库
# 然后关联远程仓库（替换成你的仓库地址）
git remote add origin https://github.com/你的用户名/你的仓库名.git

# 推送到 GitHub
git push -u origin master
```

---

## 第二步：部署后端到 Railway

### 2.1 注册 Railway 账号

1. 打开浏览器，访问 **https://railway.app**
2. 点击右上角 **"Start Your Project"**
3. 选择 **"Login with GitHub"**（用 GitHub 账号登录）
4. 授权 Railway 访问你的 GitHub

### 2.2 创建新项目

1. 登录后，点击 **"+ New Project"**
2. 选择 **"Deploy from GitHub repo"**
3. 选择你的 `guess-my-drawing` 仓库
4. Railway 会自动检测到这是 Node.js 项目

### 2.3 配置环境变量（非常重要！）

在 Railway 项目页面：

1. 点击你的项目名称
2. 点击 **"Variables"** 标签
3. 点击 **"+ New Variable"**，添加以下变量：

```
变量名：NODE_ENV
变量值：production

变量名：PORT
变量值：3000

变量名：USE_AI_WORDS
变量值：false

变量名：CORS_ORIGIN
变量值：https://你的前端域名.vercel.app
（这个等前端部署完再回来填，先留空或填临时值）
```

### 2.4 设置启动命令

1. 点击 **"Settings"** 标签
2. 找到 **"Build"** 部分
3. 在 **"Start Command"** 填入：`npm run server`

### 2.5 等待部署完成

- Railway 会自动安装依赖、构建、启动
- 查看部署日志，确认没有错误
- 通常需要 2-3 分钟

### 2.6 获取后端 URL

1. 点击 **"Settings"** 标签
2. 找到 **"Domains"** 部分
3. 点击 **"Generate Domain"**
4. 你会得到一个 URL，例如：`https://guess-my-drawing-production.up.railway.app`

**复制这个 URL！后面会用到！**

### 2.7 测试后端是否正常

在浏览器中访问：`https://你的后端URL/api/health`

应该看到：
```json
{"status":"ok","timestamp":"2026-04-28T..."}
```

## 第三步：部署前端到 Vercel

### 3.1 注册 Vercel 账号

1. 打开浏览器，访问 **https://vercel.com**
2. 点击 **"Sign Up"**
3. 选择 **"Continue with GitHub"**
4. 授权 Vercel 访问你的 GitHub

### 3.2 导入项目

1. 登录后，点击 **"Add New..."** → **"Project"**
2. 在列表中找到你的 `guess-my-drawing` 仓库
3. 点击 **"Import"**

### 3.3 配置项目

在项目配置页面：

**Framework Preset:**
- 选择 **"Vite"**（应该自动检测到）

**Root Directory:**
- 点击 **"Edit"**
- 输入：`client`
- （因为前端代码在 `client` 文件夹）

**Build Command:**
- 自动填充：`npm run build`
- 保持默认

**Output Directory:**
- 输入：`dist`
- （Vite 构建输出目录）

**Install Command:**
- 自动填充：`npm install`
- 保持默认

### 3.4 配置环境变量

在项目配置页面下方，找到 **"Environment Variables"**：

点击 **"Add Variable"**，添加：

```
名称：VITE_API_URL
值：https://你的后端URL.up.railway.app

名称：VITE_SOCKET_URL
值：https://你的后端URL.up.railway.app
```

**注意：** 把 `你的后端URL` 替换成你在第二步获取的 Railway URL

### 3.5 开始部署

1. 点击 **"Deploy"** 按钮
2. 等待部署完成（通常 1-2 分钟）
3. 看到庆祝动画 🎉 表示部署成功

### 3.6 获取前端 URL

部署成功后，Vercel 会给你一个 URL，例如：
`https://guess-my-drawing.vercel.app`

**复制这个 URL！**

---

## 第四步：更新后端 CORS 配置

### 4.1 回到 Railway

1. 打开你的 Railway 项目
2. 点击 **"Variables"** 标签
3. 找到 `CORS_ORIGIN` 变量
4. 点击编辑，修改为你的 **Vercel 前端 URL**

```
CORS_ORIGIN=https://你的前端域名.vercel.app
```

### 4.2 重新部署后端

修改环境变量后，Railway 会自动重新部署。

等待重新部署完成（通常 30 秒）。

---

## 第五步：测试部署结果

### 5.1 测试后端 API

在浏览器访问：
```
https://你的后端URL/api/health
```

应该返回：
```json
{"status":"ok","timestamp":"..."}
```

### 5.2 测试前端

在浏览器访问你的 Vercel URL：
```
https://你的前端URL.vercel.app
```

应该能看到游戏界面。

### 5.3 测试完整功能

1. **创建房间**：输入昵称，点击"创建房间"
2. **加入房间**：在另一个浏览器（或隐身模式）访问同一 URL，输入房间ID加入
3. **开始游戏**：测试游戏是否能正常运行

---

## 第六步：自定义域名（可选）

### 6.1 在 Vercel 添加自定义域名

1. 在 Vercel 项目页面，点击 **"Settings"**
2. 点击 **"Domains"**
3. 输入你的域名，如：`game.yourdomain.com`
4. 按照提示在你的域名服务商添加 DNS 记录

### 6.2 更新环境变量

在 Railway 和 Vercel 中更新环境变量，使用你的自定义域名。

## 🔧 常见问题解决

### 问题 1：前端访问显示空白页面

**原因**：环境变量配置错误

**解决**：
1. 检查 Vercel 的 `VITE_API_URL` 和 `VITE_SOCKET_URL` 是否正确
2. 确保包含了 `https://` 前缀
3. 重新部署前端

### 问题 2：Socket 连接失败

**原因**：CORS 配置错误

**解决**：
1. 检查 Railway 的 `CORS_ORIGIN` 是否包含你的前端 URL
2. 确保 URL 正确，没有多余的空格
3. 查看浏览器控制台的错误信息

### 问题 3：后端启动失败

**原因**：依赖安装失败或启动命令错误

**解决**：
1. 查看 Railway 部署日志
2. 确认 `package.json` 中的 `scripts` 有 `server` 命令
3. 确认 Railway 的 Start Command 设置正确

## 💡 提示

1. **部署顺序**：先部署后端，再部署前端，最后更新 CORS
2. **日志查看**：遇到问题先查看部署日志
3. **环境变量**：确保所有 URL 都包含 `https://` 前缀
4. **重新部署**：修改环境变量后，服务会自动重新部署
